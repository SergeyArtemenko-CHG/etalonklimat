Пользователь откатил изменения. При повторном внесении **проверять работу на сервере после каждого шага**.

**Архитектура:** /api/chat-replies проксируется Nginx на check_api.js (порт 3001). Не менять check_api.js. Логика ответов — только на фронте; при пустом ответе API не трогать стейт сообщений. Подробно: `.cursor/CHAT_AND_API_RULES.md`.

## 3. API и отображение ответов Telegram

### 3.1 chat-replies route.ts — Content-Type
- Все ответы: `Content-Type: application/json; charset=utf-8` вместо `text/plain`

### 3.2 FloatingContactBtn — fetchReplies
- Добавить `sessionId` в зависимости effect
- Guard: не запускать без sessionId (`if (!sid) return`)
- `currentSession` с fallback: `sessionIdRef.current || sessionId`

### 3.3 Сохранение при добавлении ответов
- В `setMessages` при добавлении `newReplies` сразу вызывать `saveMessages(next)` внутри коллбэка
- ID ответов: `"r-" + Math.random().toString(36).slice(2)`
