# Правила: чат-виджет и API chat-replies

## Архитектура

- **check_api.js** — работает на порту **3001**. Файл в корне проекта.
- **Nginx** — все запросы к **/api/chat-replies** проксируются на порт 3001 в check_api.js.
- Логика ответов чата **не в Next.js API** (роут Next.js для chat-replies не используется в проде). Не искать и не переписывать логику в Next.js для этого эндпоинта.

## Ограничения

1. **Никогда не изменять файл check_api.js.**
2. Работа по чату на фронте: только компонент чата (FloatingContactBtn и запросы к /api/chat-replies).
3. **При любых правках фронтенда:** если API /api/chat-replies возвращает пустой ответ (или ошибку) — **не очищать и не сбрасывать стейт сообщений**, сохранять предыдущее состояние.

## Фронтенд

- Запросы чата идут на **/api/chat-replies** (Nginx перенаправляет на 3001).
- В запросе передавать идентификатор сессии (chatId/sessionId), сохранённый при первом сообщении.
- При пустом или ошибочном ответе: не вызывать setMessages с пустым массивом; не перезаписывать messages.
